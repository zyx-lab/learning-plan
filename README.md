# learning-plan

交互式学习 skill。它把一个学习主题拆成连续文章，并根据用户在上一篇文章末尾留下的真实反馈，调整下一篇的深度、角度和节奏。

## 适用场景

- 用户想系统学习一个主题，例如「带我学奥派经济学」。
- 用户想让 AI 连续生成课程文章。
- 用户已经读完上一篇，希望根据反馈继续下一篇。
- 用户提到「下一篇」「学习反馈」「继续学」「带我学」。

## 触发方式

常用触发词：

- `/learning-plan`
- `/learn`
- `/交互式学习`
- `带我学一个课题`
- `继续下一篇`
- `根据我的反馈写下一篇`

英文触发也可用：

- `teach me a topic`
- `continue the next lesson`

## 核心能力

- 创建连续学习课题目录和学习计划。
- 生成第 1 篇学习文章，并在末尾留下「学习反馈」区域。
- 读取上一篇文章和用户反馈，再决定下一篇的难度与方向。
- 根据反馈放慢节奏、补例子、转向应用、提高概念密度或优先回答问题。
- 维护 `00-学习计划.md` 和可选的 `INDEX.md` 索引。

## 文件结构

每个课题目录固定使用以下结构：

```text
{课题名}/
├── 00-学习计划.md
├── 01.md
├── 02.md
├── 03.md
└── assets/
```

命名规则：

- 学习计划固定为 `00-学习计划.md`。
- 学习文章使用两位数字序号，例如 `01.md`、`02.md`、`03.md`。
- 下一篇序号取当前课题目录中最大文章序号加 1。
- 文章文件名不使用中文标题，不跳号。

## 存放规则

目录选择优先级：

1. 用户明确指定目录时，使用用户指定目录。
2. 当前目录是项目目录时，使用 `当前目录/学习课题/{课题名}/`。
3. 当前目录是泛目录或系统目录时，使用 `~/Documents/dbskill-learning/{课题名}/`。

项目目录判断依据：当前目录中存在 `.git`、`README.md`、`AGENTS.md`、`CLAUDE.md`、`package.json`、`pyproject.toml`、`知识库/` 或 `skills/` 中任意一项。

全局索引默认位置：

```text
~/Documents/dbskill-learning/INDEX.md
```

项目内保存学习文件时，可维护：

```text
学习课题/INDEX.md
```

## 工作流程

### 1. 确认课题

用户没有给出课题时，先询问想系统学习哪个主题。用户给出课题后，确认课题名和存放目录。

新课题会创建：

- 课题目录
- `00-学习计划.md`
- `01.md`
- `assets/`
- 索引记录

已有课题会读取现有进度并继续。

### 2. 读取进度

生成下一篇前必须读取：

- `00-学习计划.md`
- 当前课题目录中序号最大的文章
- 文章末尾「学习反馈」区域中用户实际填写的内容
- 课题目录内其他可能包含反馈的文件

反馈区里的默认提示问题会被忽略。过滤模板行后没有新增内容时，视为没有反馈。

### 3. 判断学习梯度

根据用户反馈调整下一篇：

| 用户反馈信号 | 下一篇处理方式 |
|---|---|
| 没看懂、概念混乱、问题很多 | 降低抽象度，补例子，放慢节奏 |
| 看懂了但觉得没意思 | 换切入角度，连接用户真实问题 |
| 看懂了并提出应用问题 | 增加案例、判断方法和使用场景 |
| 明显掌握了 | 提高概念密度，进入下一层 |
| 提出具体问题 | 优先回应问题，再推进课程 |
| 反馈很少 | 保持当前难度，小步推进 |

### 4. 生成下一篇

学习文章固定包含：

```markdown
# {序号}｜{标题}

## 这一篇要解决的问题

## 正文

## 小结

## 下一篇预告

---

## 学习反馈
```

「学习反馈」区域会保留 4 个提示问题，方便用户写下理解、困惑、延展方向和真实应用场景。

### 5. 更新计划和索引

生成文章后更新 `00-学习计划.md`：

- 当前进度
- 本篇主题
- 用户上一轮反馈摘要
- 下一篇方向
- 最近更新时间

使用 `INDEX.md` 时，同步更新课题、当前进度、最近更新和下一步。

## 写作原则

- 始终使用中文。
- 表达清晰、有深度，像懂行的朋友在讲解。
- 直接讲清楚机制，避免空洞的教科书腔调。
- 不预判读者错误，不用居高临下的纠错姿态。
- 中英文之间加空格，中文与数字之间加空格，中文标点使用全角，数字使用半角。

## 验收用例

### 新课题

输入：「带我学奥派经济学。」

应完成：

- 确定课题目录。
- 创建 `00-学习计划.md`。
- 创建 `01.md`。
- 在 `01.md` 末尾包含「学习反馈」区域。

### 反馈没看懂

用户在 `01.md` 末尾写：「我没看懂供需曲线。」

应完成：

- 读取 `01.md`。
- 提取用户真实反馈。
- 在 `02.md` 降低抽象度，用更具体的例子解释。
- 暂缓堆叠新概念。

### 反馈想应用

用户在 `01.md` 末尾写：「这个我懂了，我更想知道它怎么用于商业判断。」

应完成：

- 读取 `01.md`。
- 提取用户真实反馈。
- 在 `02.md` 转向案例和判断方法。
- 保持和原课题的连续性。
